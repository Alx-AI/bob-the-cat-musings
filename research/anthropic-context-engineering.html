Effective Context Engineering for AI Agents

Anthropic ¬∑ Sep 29, 2025
Source: anthropic.com/engineering

Context engineering is the art and science of curating what will go into the limited context window from a constantly evolving universe of possible information.

Context Engineering > Prompt Engineering

Prompt engineering = how to write effective prompts, especially system prompts. Optimized for one-shot tasks.

Context engineering = curating the optimal set of tokens during inference ‚Äî system instructions, tools, MCP, external data, message history, everything. Iterative. Curation happens each turn.

As agents run longer loops generating more data, managing the whole state matters more than nailing one prompt.

Why Context Is Finite

The key concept: context rot. As token count increases, the model's ability to recall information decreases.

  • Transformer architecture: every token attends to every other ‚Üí n¬≤ pairwise relationships
  • As context grows, ability to capture these relationships stretches thin
  • Models trained more on shorter sequences ‚Üí less experience with long-range dependencies
  • It's a performance gradient, not a cliff ‚Äî but precision degrades

Think of it as an "attention budget" that depletes with each token. Every piece of information you add has a cost.

The Goldilocks Zone for System Prompts

Two failure modes:

  • Too specific: hardcoded complex brittle logic ‚Üí fragility, maintenance nightmare
  • Too vague: high-level hand-waving that fails to give concrete signals

The sweet spot: specific enough to guide behavior effectively, flexible enough to give the model strong heuristics. Organize with XML tags or Markdown headers. Start minimal, add based on observed failures.

"Just In Time" Context

Instead of pre-loading all relevant data upfront:

  • Maintain lightweight identifiers: file paths, stored queries, web links
  • Load data dynamically at runtime using tools
  • Claude Code writes targeted queries, uses head and tail to analyze large databases without loading everything

This mirrors human cognition ‚Äî we maintain filing systems and bookmarks, not memorized corpuses.

Progressive Disclosure

Let agents discover context incrementally through exploration. Metadata provides signals:

  • test_utils.py in tests/ vs src/core_logic/ implies different purposes
  • Folder hierarchies, naming conventions, timestamps all help

Tool Design for Context Efficiency

  • Tools should be self-contained, robust to error, clear about intended use
  • Most common failure: bloated tool sets with too much functionality or ambiguous overlaps
  • If a human can't say which tool to use, an agent can't either
  • Curate a minimal viable tool set

On Examples

Don't stuff a laundry list of edge cases. Curate diverse, canonical examples that portray expected behavior. For LLMs, examples are "the pictures worth a thousand words."
