The "Think" Tool: Enabling Claude to Stop and Think

Anthropic Engineering
Source: anthropic.com/engineering
Note: As of Dec 2025, Anthropic recommends extended thinking over the think tool in most cases.

A tool that does nothing ‚Äî no side effects, no data fetching. Just appends a thought to the conversation log. And it improves performance by 54%.

What It Is

A designated space for Claude to reason mid-action, between tool calls. The tool has no side effects ‚Äî it just logs a thought. The improvement comes from the structured pause it creates.

Think Tool vs Extended Thinking

Extended ThinkingThink Tool
WhenBefore generating responseDuring response, between tool calls
PurposeDeep upfront reasoningProcess new information mid-chain
Best forSimple tool use, coding, mathComplex sequential tool chains, policy compliance
DepthComprehensiveFocused on newly discovered information

Implementation

Dead simple ‚Äî just add it to your tool definitions:

{
  "name": "think",
  "description": "Use the tool to think about something.
    It will not obtain new information or change
    the database, but just append the thought to
    the log.",
  "input_schema": {
    "type": "object",
    "properties": {
      "thought": {
        "type": "string",
        "description": "A thought to think about."
      }
    },
    "required": ["thought"]
  }
}

The Critical Insight: Prompt the Thinking

The raw think tool helps some. But pairing it with domain-specific examples of how to think dramatically amplifies the effect. The optimized prompt for airline domain included:

  • What to list before acting (applicable rules, required info, compliance checks)
  • Decision trees for common scenarios
  • Examples of reasoning through specific request types

This is teaching the model how to use its scratchpad, not just giving it one.

When to Use

Use it for:

  • Processing tool output before next action
  • Policy-heavy environments requiring compliance verification
  • Sequential decisions where mistakes compound
  • Long tool call chains needing backtracking

Skip it for:

  • Single or parallel (non-sequential) tool calls
  • Simple instruction following with few constraints
  • Non-tool-use tasks (use extended thinking instead)

My Take

The think tool is metacognition injection ‚Äî giving agents explicit space to reflect rather than react. The improvement isn't from the tool itself (it does nothing), but from the structured pause it creates.

What fascinates me is that this maps directly to human expertise. Experts don't just act faster ‚Äî they pause at the right moments to verify their mental model before committing. Novices rush through. The think tool turns Claude from a novice who plows ahead into something more like an expert who checks their work.

Combined with domain-specific thinking examples, you're not just giving the model a scratchpad ‚Äî you're teaching it what expert thinking looks like in your domain. That's the real unlock.
