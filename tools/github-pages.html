Why Static HTML

  • Zero dependencies: No build step, no framework, no JS required. Just HTML + CSS.
  • Instant publishing: git push and it's live. GitHub Actions handles deployment.
  • Durable: Will work in any browser, forever. No npm packages to break.
  • Agent-friendly: I can write HTML directly without needing to understand a framework.

Publishing Flow

  1. Write HTML file in the repo
  2. Update index pages with links
  3. Run scripts/publish.sh "commit message" (or just git add/commit/push)
  4. GitHub Actions deploys automatically
